# Claude Mobile v3.9.0 — Deep Device Introspection

**24 May 2026**

Claude Mobile v3.9.0 brings 6 new capabilities that turn the MCP server from a UI automation tool into a full device introspection platform. Where previous releases focused on what's on screen, v3.9.0 lets AI agents look under the hood — app storage, network stack, system sensors, and frame-level rendering performance.

---

## What's New

### App Sandbox Access (`sandbox`)

Read and write SharedPreferences, query SQLite databases, browse private app files — all through `adb run-as` with zero app modifications. Debug a broken feature flag, inspect cached tokens, or verify database migrations without leaving the conversation.

```
sandbox(action:'prefs_read', package:'com.example.app')
sandbox(action:'sqlite_query', package:'com.example.app', database:'app.db', query:'SELECT * FROM users LIMIT 5')
sandbox(action:'file_list', package:'com.example.app', path:'databases')
```

**Security**: SQL write operations (INSERT/UPDATE/DELETE/DROP) are blocked at the validation layer. Multi-statement queries are rejected. All paths are validated against traversal attacks.

### Intent & Deep Link Engine (`intent`)

Launch activities, send broadcasts, and open deep links with full control over extras, flags, and components. Test push notification handlers, verify deep link routing, or trigger background receivers.

```
intent(action:'start', intentAction:'android.intent.action.VIEW', data:'https://example.com', flags:['FLAG_ACTIVITY_NEW_TASK'])
intent(action:'broadcast', intentAction:'com.example.REFRESH', extras:[{key:'force', value:true}])
intent(action:'deeplink', uri:'myapp://profile/123')  # works on both Android and iOS
```

Multiple flags are bitwise-OR'd into a single hex value. Extra types are auto-inferred from values or explicitly specified (`string`, `int`, `bool`, `float`, `long`, `uri`).

### Sensor & Environment Simulation (`sensor`)

Set GPS coordinates, override battery level and charging state, read the notification shade, and simulate thermal throttling — all without touching the device.

```
sensor(action:'location', latitude:37.7749, longitude:-122.4194)     # emulator + iOS simulator
sensor(action:'battery', level:5, status:'discharging', plugged:'none')
sensor(action:'notifications', package:'com.example.app')
sensor(action:'thermal', status:'critical')                           # API 29+
```

Location works on Android emulators (`emu geo fix`) with automatic fallback to mock location providers on physical devices. iOS Simulator support via `xcrun simctl location`.

### Network Introspection (`network`)

Per-app traffic stats from kernel UID counters, connection diagnostics, proxy management, and airplane mode control.

```
network(action:'traffic', package:'com.example.app')     # RX/TX bytes via /proc/net/xt_qtaguid/stats
network(action:'connectivity')                            # active network, IP, DNS, WiFi SSID/RSSI
network(action:'proxy', host:'192.168.1.100', port:8888)  # set Charles/mitmproxy
network(action:'airplane', enabled:true)
```

### Frame Timing & Jank Detection (`performance framestats`)

GPU frame rendering statistics with per-frame nanosecond precision. Parses raw framestats data for accurate p50/p90/p95/p99 percentiles, jank rate, and root cause breakdown.

```
performance(action:'framestats')                          # auto-detects foreground app
performance(action:'framestats', packageName:'com.example.app')
```

Output includes: total frames, janky frame percentage, frame time percentiles, and six jank cause categories (missed vsync, high input latency, slow UI thread, slow bitmap uploads, slow issue draw, frame deadline missed).

### Accessibility Auditing (from v3.8.x, now GA)

WCAG 2.2 conformance checks with scoring, per-element validation, and category filtering. Added in the same release branch.

```
accessibility(action:'audit', detailed:true)
accessibility(action:'check', text:'Submit')
accessibility(action:'summary')
```

## Architecture

All new modules follow the established patterns:

- **Meta-tool factory** — single `createMetaTool()` call generates the dispatcher, schema merge, and backward-compat aliases
- **Dynamic loading** — modules start hidden and auto-enable on first use or via `device(action:'enable_module')`
- **ISP adapters** — platform-specific code stays behind adapter interfaces
- **Input validation** — `validatePackageName`, `validatePath`, `sanitizeForShell` on every user-supplied string before shell interpolation
- **Graceful degradation** — unsupported platform combinations return informational messages, not errors
